pay.json — machine-readable pricing for AI agents
Like robots.txt but for payments. Placed at /.well-known/pay.json on your site. Xenarch generates it automatically.
pay.json
myblog.com/.well-known/pay.json
{
  "version": "1.0",
  "provider": "xenarch",
  "protocol": "x402",
  "network": "base",
  "asset": "USDC",
  "receiver": "0x4a2E...8f3D",
  "seller_wallet": "0x9b2e...f104",
  "facilitator": "https://api.xenarch.dev/v1/gates",
  "rules": [
    { "path": "/*", "price_usd": "0.003" },
    { "path": "/docs/*", "price_usd": "0.01" },
    { "path": "/api/*", "price_usd": "0.05" },
    { "path": "/public/*", "price_usd": "0" } // free
  ],
  "contact": "hello@myblog.com",
  "terms": "https://myblog.com/terms"
}
What it does
Tells AI bots: "this site charges for access." Bots check it before crawling, just like they check robots.txt.
Rules
Your pricing from the Settings tab, exported as machine-readable JSON. First match wins. Price "0" = free access.
Auto-generated
Xenarch creates and serves this file automatically based on your pricing rules. No manual editing needed.
Open standard
pay.json is a vendor-neutral spec. Any payment provider can implement it. Not locked to Xenarch.
